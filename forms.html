Form Structure

  • From controls always live nside Frm element, this element always cary the acction attriubte and usualy will have a method and id attibute.
  • Every form element requires an Action attribute. Its value is the URL for the page on the server that will receive the information.
  • Froms can be sent using oe of tow methods: get or post
    1. With the get method, the values from the form are added to the end of the URL specified in the action attribute. the get method is ideal for:
      • short forms (such as search boxes)
      • when you are just retrieving data from the web server (not sending information that shoud be added to or deleted from a database)
    2. With post method the values are sent in what are knows as HTP headers. As rule of thumb you shoud use the Post method if your for:
      • allow users to upload a file
      • is very long
      • contains senstive data(e.g. passwords)
      • adds information to, or deletes information from, a database
  • If the method attribute is not used, the data will be sent using the get method
  • We look at the id attribute.. but the value is used to identify the from distinctly from other elements on the page(and is often used by scripts--such as those that check you have entered information into fields that require values).

(for example, in a login form, the server needs to know what has been entered as the username and what hassben given as the password.) Therefore, eachform control requires a name attribute. The value of ths attirbute identifies the form control and is sent along with the information they enter the server.
